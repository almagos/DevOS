# Documentation

## Purpose

Serves as the root navigation entry point for all canonical repository documentation and knowledge domains.

## Contains

Top-level knowledge domains for the repository:
- `product/`: Product strategy, domain, users, features, and integrations.
- `engineering/`: System architecture, platform, repository structure, design patterns, and ADRs.
- `operations/`: Release processes, monitoring, and operational runbooks.
- `playbooks/`: Repeatable workflow definitions and playbooks.
- `standards/`: Constraints, standards, and guidelines.
- `templates/`: Reusable document templates.

## Does Not Contain

- Implementation code.
- Ephemeral planning, roadmaps, or issue tracking data.
- Generated operational state or temporary AI artifacts (stored under `.devos/`).

## Related Directories

- `product/`
- `engineering/`
- `operations/`
- `playbooks/`
- `standards/`
- `templates/`

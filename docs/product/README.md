# Product

## Purpose

Defines product-level knowledge explaining what the system is, who it serves, and how features and integrations align.

## Contains

- `strategy/`: Product strategy, vision, and roadmap alignment.
- `domain/`: Core domain models, business logic boundaries, and terminology.
- `users/`: Target personas, user journeys, and user requirements.
- `features/`: Functional feature specifications and behavior models.
- `integrations/`: External product and service integration points.

## Does Not Contain

- Engineering implementation details or code architecture.
- Operational runbooks or deployment instructions.
- Work tracking issues or unvalidated feature requests.

## Related Directories

- `docs/engineering/`
- `docs/operations/`

Subdirectories and markdown files must use lower-case kebab-case (e.g., `user-personas.md`).

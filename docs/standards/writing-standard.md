# Writing Standard

> **Status:** Stable
>
> **Version:** 1.0
>
> **Owner:** DevOS
>
> **Applies To:** Repository Documentation
>
> This standard defines the writing conventions used throughout the repository.
>
> All documentation should read as if written by a single author.

---

# 1. Purpose

The purpose of this standard is to ensure repository documentation remains:

- Clear
- Consistent
- Precise
- Unambiguous
- AI-friendly

Writing quality is considered part of repository quality.

---

# 2. Scope

This standard applies to:

- AGENTS.md
- Documentation
- Playbooks
- Standards
- Templates
- Repository documentation

It does not apply to:

- Source code
- Generated operational artifacts

---

# 3. Writing Principles

All documentation follows these principles.

## Principle 1

Write in the present tense.

---

## Principle 2

Use active voice whenever possible.

---

## Principle 3

Be concise.

Avoid unnecessary explanation.

---

## Principle 4

Be explicit.

Avoid ambiguous language.

---

## Principle 5

Use stable terminology.

The same concept should always use the same name.

---

## Principle 6

Write for both humans and AI agents.

---

# 4. Language

Repository documentation should:

- Use complete sentences.
- Use consistent terminology.
- Prefer simple language.
- Avoid conversational wording.

---

# 5. Terminology

Repository terminology should remain consistent.

Examples include:

- Canonical Knowledge
- Knowledge Synchronization
- Parent Issue
- Child Issue
- Repository Constitution
- Generated Operational State

Alternative names should not be introduced unless repository terminology is intentionally revised.

---

# 6. Structure

Documents should:

- Have a single responsibility.
- Use consistent section headings.
- Follow approved templates where applicable.

---

# 7. Prohibited Content

Repository documentation should avoid:

- Implementation history
- Temporary planning notes
- Brainstorming
- Personal opinions
- AI-generated reasoning
- Duplicate explanations

---

# 8. Compliance

Documentation complies with this standard when:

- Terminology is consistent.
- Writing is concise.
- Language is unambiguous.
- Repository conventions are followed.

---

# 9. Exceptions

Exceptions should be rare.

They should improve clarity without reducing consistency.

# Engineering

## Purpose

Serves as the knowledge hub for technical design, software architecture, platform infrastructure, and engineering decisions.

## Contains

- `architecture/`: High-level system structure and subsystem design.
- `platform/`: Infrastructure, hosting, CI/CD, and environment specifications.
- `repository/`: Documentation architecture, code organization rules, and repository guidelines.
- `patterns/`: Approved design patterns, coding practices, and architectural conventions.
- `decisions/`: Architectural Decision Records (ADRs) capturing key architectural choices.

## Does Not Contain

- Product strategy or user personas.
- Live secrets or environment variables.
- One-off debug logs or temporary local scripts.

## Related Directories

- `docs/product/`
- `docs/operations/`
- `docs/standards/`

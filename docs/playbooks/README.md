# Playbooks

## Purpose

Defines repeatable execution workflows and step-by-step procedures for human and AI engineering tasks.

## Contains

- Standard operating procedures for recurring engineering tasks.
- Guided execution playbooks (e.g., knowledge synchronization, refactoring).
- Process workflows independent of specific tools.

## Does Not Contain

- Static policy rules or code formatting constraints (see `docs/standards/`).
- Specific product requirements.
- Temporary scratch notes.

## Related Directories

- `docs/standards/`
- `docs/templates/`
- `docs/operations/runbooks/`

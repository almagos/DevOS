## Purpose

DevOS is a Development Operating System that provides a consistent way to plan, execute, review, document, and evolve software projects.

It defines how work moves from ideas to production while ensuring that knowledge, implementation, and governance remain synchronized.

---

## Vision

To establish a reusable development operating system that enables humans and AI agents to build software using a common set of principles, workflows, and standards.

DevOS should be reusable across every Almagos project without redesigning the development process.

---

## Mission

DevOS enables repeatable software development by:

- Separating planning from implementation.
- Separating work management from knowledge management.
- Standardizing development workflows.
- Keeping documentation synchronized with implementation.
- Supporting interchangeable engineering executors.

---

## Goals

DevOS aims to:

- Standardize software development.
- Reduce development friction.
- Preserve architectural knowledge.
- Make work traceable.
- Keep documentation current.
- Enable collaboration between humans and AI.
- Allow projects to reuse proven workflows.

---

## Guiding Principles

### Execution over Speculation

Architecture is validated through execution, not endless discussion.

Approved decisions are implemented before being reconsidered.

---

### Knowledge is Permanent

Approved knowledge belongs in the repository.

Conversations are temporary reasoning spaces.

---

### Work is Traceable

Every significant change is represented by a work item.

---

### One Issue → One Primary Pull Request

Every work item produces one primary Pull Request containing every artifact modified by that work.

---

### Documentation Evolves with Implementation

Documentation changes are part of the same work item whenever behavior changes.

---

### Roles over Tools

Responsibilities belong to roles.

Tools and AI systems are interchangeable executors.

---

### Standards before Implementation

Implementation follows approved standards.

Standards evolve through the same workflow used for products.

---

### Simplicity over Abstraction

DevOS adopts the terminology of its primary platforms unless abstraction provides clear long-term value.

---

## Scope

DevOS governs:

- Development workflow
- Work management
- Knowledge management
- Repository organization
- Documentation
- Engineering workflow
- Review workflow
- Release workflow

---

## Out of Scope

DevOS does not define:

- Product requirements
- Business strategy
- Programming languages
- Frameworks
- Infrastructure
- UI or UX design
- Technology choices

These belong to individual projects.

---

## Success Criteria

DevOS is successful when:

- New projects adopt the operating system without redesign.
- Documentation and implementation remain synchronized.
- Development is repeatable across projects.
- AI and human executors can collaborate through the same operating model.
- Knowledge remains discoverable and version controlled.
